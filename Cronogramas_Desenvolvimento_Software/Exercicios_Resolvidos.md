--------------------------------------------------

A) Projeto: Sistema Hospitalar

Possíveis riscos que podem comprometer o cronograma:

1. Mudanças frequentes nos requisitos do sistema;
2. Falta de comunicação entre equipe e cliente;
3. Atrasos na modelagem do banco de dados;
4. Problemas na integração entre módulos;
5. Sobrecarga dos desenvolvedores;
6. Retrabalho causado por erros encontrados nos testes;
7. Dependência de sistemas externos;
8. Falta de documentação adequada;
9. Problemas de infraestrutura;
10. Complexidade excessiva do sistema hospitalar.

Além disso, a utilização incorreta de padrões de projeto pode gerar dificuldades de manutenção e aumentar o tempo de desenvolvimento.

Um exemplo importante é o Factory Method, apresentado no site Refactoring Guru, que ajuda na criação de objetos de forma organizada e desacoplada, facilitando manutenção, escalabilidade e reutilização do código.

Link de referência:
https://refactoring.guru/pt-br/design-patterns/factory-method

B) Cronograma Completo - Sistema Web Hospitalar

Projeto: Sistema Hospitalar Web

1. Reunião inicial com cliente - 1 dia
2. Levantamento de requisitos - 5 dias
3. Análise do sistema - 4 dias
4. Modelagem UML - 3 dias
5. Modelagem do banco de dados - 3 dias
6. Planejamento da arquitetura - 2 dias
7. Protótipo das telas - 4 dias
8. Aprovação do protótipo - 1 dia
9. Desenvolvimento do login - 3 dias
10. Desenvolvimento do cadastro de pacientes - 4 dias
11. Desenvolvimento do agendamento - 5 dias
12. Desenvolvimento do prontuário eletrônico - 6 dias
13. Desenvolvimento frontend - 6 dias
14. Integração dos módulos - 4 dias
15. Testes unitários - 3 dias
16. Testes de integração - 3 dias
17. Homologação - 2 dias
18. Implantação - 2 dias

--------------------------------------------------

Dependências:

- A análise depende do levantamento de requisitos;
- O banco de dados depende da modelagem;
- O desenvolvimento depende da aprovação do protótipo;
- Os testes dependem da integração;
- A implantação depende da homologação.

--------------------------------------------------

Marcos do projeto:

- Requisitos aprovados;
- Banco de dados concluído;
- Protótipo aprovado;
- Sistema integrado;
- Testes finalizados;
- Sistema implantado.

--------------------------------------------------

Caminho crítico:

Levantamento de requisitos
→ Análise
→ Banco de dados
→ Desenvolvimento
→ Integração
→ Testes
→ Implantação

Conclusão:

O cronograma permite organizar as atividades do projeto, controlar prazos e identificar riscos que podem impactar o desenvolvimento do software.

A utilização de boas práticas de engenharia de software e padrões de projeto, como o Factory Method, contribui para um sistema mais organizado, reutilizável e de fácil manutenção.
